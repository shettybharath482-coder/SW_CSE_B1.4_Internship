CSS Units: are the measurement units used to specify sizes, spacing & length related values in properties like width, height, margin etc. How big or small the elements are going to be.

1. Absolute Units: represent fixed values. They don't change based on parent element, screen size or font-size. >> Pixels (px): 1px is considered as 1/96 of an inch >> Centimeters (cm), millimeters (mm), inches(in): Basis of real-world measurements >> Points (pt), picas(pc): legacy approach When to use?: For print layout

2. Relative Units: are based on relation with another element's size, viewport size. Responsive designs. a) Font-related units: >> em: relative to the font-size of the current element's parent >> rem: root em: relative to the root element's(html) font size, not the parent >> ex: reletive to the x-height b) Percentage based units: >> %(percent): relative to element's parent c) viewport-relative units: viewport: is the visible area of a web page in the browser window or on a device screen. The rectangular area/region where browser renders and shows content. >> vw: viewport width 1 vw = 1% of the viewport width >> vh: viewport height 1 vw = 1% of the viewport height >> vmin: 1 vmin = 1% of the smaller dimension (width or height) of the viewport >> vmax: 1 vmax = 1% of the larger dimension (width or height) of the viewport When to use which unit?: >> px: Fine or precise values >> rem: Font sizes or whenver spacing is needed, Responsive >> %: Great width or heights >> vw,vh,vmin, vmax: Ideal for full-screen sections, it adapts to screen size >> cm,mm,in,pt: For print layout
